# 기여 가이드

## 원칙

- 모든 Markdown 문서는 한글 본문을 기본으로 작성합니다.
- 개인 계정, 개인 도메인, 실제 앱 Bundle ID, secret 경로를 하드코딩하지 않습니다.
- 새 스크립트는 환경변수 기반으로 작성하고 파일 상단 또는 관련 README에 필수 값을 문서화합니다.
- Terraform은 AWS 선택 시에만 `infra/aws/` 아래에 둡니다.

## 변경 전 확인

```sh
npm run ci
```

CI는 문서 규칙, 민감정보 패턴, JavaScript 문법, shell 문법, Terraform 포맷을 확인합니다.

## Pull Request 기준

- 새 기능은 README 또는 관련 하위 문서에 사용법을 포함합니다.
- 배포 관련 변경은 환경변수 누락 시 사용자가 무엇을 설정해야 하는지 안내해야 합니다.
- 앱 소스 예제는 기본적으로 추가하지 않습니다. 템플릿은 가볍게 유지합니다.
